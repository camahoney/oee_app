VSM
Continuous Improvement

Session Overview

Duration: 60-90 Minutes (Intro) / Multi-Day (Event)

Goal: To teach teams how to use VSM to make measurable changes (lead time, OEE, safety, quality), not just how to draw icons.

Preparation Checklist
  • [ ] Projector/TV for Slide Deck.
  • [ ] Prop: Butcher paper and markers (Color coded).
  • [ ] Prop: Post-it notes (Yellow for process, Pink for waste).
  • [ ] Data: Recent lead time or quality metrics (site specific).
Core Concepts

1. Value Stream vs. Process: A value stream is the end-to-end flow for a product family (order to delivery), not just a single department.

2. Current vs. Future State: Current state shows how things truly work today (warts and all); Future state is a deliberately designed flow.

3. Value vs. Non-Value Added: Distinguish processing time that changes the product from queues, waits, transport, and rework.

Instructor Emphasis

"Connect to your site metrics (injury rate, changeover time, past quality escapes). VSM is a tool to fix problems, not a generic Lean exercise."

Key Stat: Use real examples where >90% of total lead time is waiting, not working.

Module 1: The Why
Value Proposition

VSM visualizes every step of material and information flow from order to delivery so teams can see how value is (and is not) created for the customer.

Done well, it cuts lead time, reduces inventory, improves OEE, and exposes bottlenecks that daily firefighting hides.

Why this matters here:

VSM links continuous improvement to business goals by connecting problems (e.g., delay, rework, safety risk) directly to process steps and data.

Step-by-Step Application (Method)

1. Define the product/service family: Select one value stream with high business impact (volume, risk, complaints).

2. Build the current state map: Create this ON THE FLOOR, not in a conference room.

  • Walk the flow backwards (Customer -> Shipping -> Production -> Supplier).
  • Capture key data at each step: Cycle time, Changeover, Uptime/OEE, Scrap, WIP.
  • Capture Information Flow: How are orders released? Look for conflicting signals.

Data Collection

"Make sure each process has one clear production signal instead of multiple conflicting signals."

Module 2: The Future
3. Analyze Current State
  • Identify Constraints: Find bottleneck processes using measured cycle times, not estimates. Eliminate "tribal knowledge."
  • Quantify Waste: Total value-added time vs. total lead time. How much is waiting?
  • Find Systemic Problems: Overproduction, excess inventory, poor info flow, unbalanced work.
4. Design Future State

Align with business targets and customer demand (Takt time).

  • Simplify information flow (one clear signal).
  • Reduce batch sizes and queues.
  • Introduce Pull Systems, Supermarkets, and Level Scheduling.
5. Execute Implementation Plan

Translate the future state map into a prioritized set of changes (e.g., SMED on line X, standard work at step Y).

Trainer Consensus:

Every VSM event must end with a documented implementation plan and follow-up cadence; otherwise the map generates frustration instead of value.

Review
Best Practices vs Pitfalls
  • Do: Build a cross-functional team (including operators). Use real, current data gathered at the gemba.
  • Don't: Add too much detail (unreadable). Design with multiple info signals (confusing). Misidentify bottlenecks due to averaged data.
Safety & Culture Integration

VSM can surface safety issues (excessive transport, congestion) as part of the current state. Future state design can intentionally remove those risks.

Using VSM regularly builds a culture where teams see problems as system design issues, not individual blame.

Integration

Integrate VSM with existing Lean/CI systems (5S, standard work) to ensure sustainable change.

Competency Check

Q1: Why do we map the "Current State" first?

A: To see how things truly work ("warts and all") before designing a future state.

Q2: What is a "Value Stream"?

A: The end-to-end flow for a product family, covering both material and information flow.
